WFM Live Preview

This page provides a live preview for Wiki-Flavored Markdown (WFM), rendered client-side using JavaScript.

The majority of the credit goes to the wonderful Showdown script, which handles the meat of the processing, as well as to GitHub's adaptation.

Differences from Standard Markdown

wiki links

WFM supports three types of wiki links:

  • automatic detection via CamelCase: PageName
  • free links surrounded by brackets: [[page name]]
  • free links with custom link text: [[my page|page name]]
URL auto-linking
WFM will auto-link standard URLs, so if you want to link to a URL (instead of setting link text), you can simply enter the URL and it will be turned into a link to that URL.
fenced code blocks
Markdown converts text with four spaces at the front of each line to code blocks. WFM supports that, but it also supports fenced blocks. Just wrap your code blocks in ``` and you won't need to indent manually to trigger a code block.
multiple underscores in words
In order to avoid confusion, italicizing just parts of a word is not possible; WFM ignores multiple underscores in words.
